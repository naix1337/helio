Architektur-Optionen: Proxmox Auto-Detection

Wie soll der Connector arbeiten? Klicke auf eine Option.

A

Backend-Connector (empfohlen)

Neues Modul ProxmoxCollector im Backend, das die Proxmox API direkt pollt. Kein separater Agent nötig. Erstellte Container/VMs landen automatisch in der Agents-Tabelle mit Metriken und Status.

Vorteile

  • Keine extra Installation
  • Nutzt bestehenden Collector-Pattern
  • Einfachste Konfiguration

Nachteile

  • Backend braucht Netzwerkzugriff auf Proxmox API
B

Helio-Agent Extension

Den bestehenden helio-agent erweitern, sodass er die Proxmox API abfragt und die Container/VM-Liste per WebSocket meldet.

Vorteile

  • Läuft auf dem Proxmox-Host selbst
  • Kein direkter Backend-Zugriff nötig

Nachteile

  • Agent muss installiert werden
  • Komplexere Daten-Synchronisation
  • Zusätzlicher Token nötig
C

Frontend-zentriert

Proxmox-Verbindung wird im Frontend konfiguriert, das Backend ruft die API on-demand (nicht polling) auf und erstellt Agents manuell.

Vorteile

  • Einfach zu implementieren

Nachteile

  • Keine automatische Erkennung
  • Keine Live-Metriken
  • Langsamer
